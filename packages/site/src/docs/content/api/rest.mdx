# REST API

The Peril server exposes a REST API for managing reviews.

## Base URL

```
http://127.0.0.1:4173/api
```

## Endpoints

### List Reviews

```http
GET /api/reviews
```

Query parameters:

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | `string` | Filter by status |
| `category` | `string` | Filter by category |
| `severity` | `string` | Filter by severity |
| `url` | `string` | Filter by page URL |
| `limit` | `number` | Maximum number of reviews to return |

### Get Review

```http
GET /api/reviews/:id
```

Returns the full review object including locators, metadata, and comment.

### Create Review

```http
POST /api/reviews
Content-Type: application/json
```

Created automatically by the SDK when a user submits feedback.

### Update Review

```http
PATCH /api/reviews/:id
Content-Type: application/json

{
  "status": "resolved",
  "resolution": {
    "resolvedAt": "2026-03-31T20:45:00.000Z",
    "resolvedBy": "codex",
    "comment": "Fixed the issue"
  }
}
```

### Get Artifact

```http
GET /api/reviews/:id/artifacts/:type
```

Returns the binary screenshot or snapshot. Type is `elementScreenshot`,
`pageScreenshot`, or `rrwebSession`.

### Delete Review

```http
DELETE /api/reviews/:id
```

### Health Check

```http
GET /api/health
```

Returns a JSON payload like:

```json
{
  "status": "ok",
  "service": "@peril-ai/server",
  "timestamp": "2026-03-31T20:45:00.000Z"
}
```
