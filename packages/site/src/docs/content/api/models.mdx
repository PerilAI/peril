# Data Models

Reference for the core data types used throughout Peril.

## Review

```typescript
interface Review {
  id: string; // ULID prefixed with rev_
  url: string;
  timestamp: string; // ISO 8601
  viewport: {
    width: number;
    height: number;
  };
  status: ReviewStatus;
  selection: {
    boundingBox: {
      x: number;
      y: number;
      width: number;
      height: number;
    };
    locators: LocatorBundle;
    domSnippet: string;
    computedStyles?: Record<string, string>;
  };
  comment: {
    category: ReviewCategory;
    severity: Severity;
    text: string;
    expected: string;
  };
  artifacts: {
    elementScreenshot: string;
    pageScreenshot: string;
    rrwebSession?: string;
  };
  resolution: {
    resolvedAt: string;
    resolvedBy: string;
    comment?: string;
  } | null;
  metadata: {
    userAgent: string;
    scrollPosition: { x: number; y: number };
    devicePixelRatio: number;
    reviewerName?: string;
  };
}
```

## LocatorBundle

```typescript
interface LocatorBundle {
  testId?: string;
  role?: {
    type: string;
    name: string;
  };
  css: string;
  xpath: string;
  text?: string;
}
```

## Enums

```typescript
type ReviewCategory = "bug" | "polish" | "accessibility" | "copy" | "ux";

type Severity = "low" | "medium" | "high" | "critical";

type ReviewStatus = "open" | "in_progress" | "resolved" | "wont_fix";
```

<Callout variant="note">
  All IDs use ULID format with a `rev_` prefix for easy identification.
  Timestamps follow ISO 8601 format.
</Callout>
