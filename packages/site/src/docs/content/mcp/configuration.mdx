# MCP Configuration

The Peril MCP server exposes review tasks to coding agents using the
Model Context Protocol.

## Starting the Server

```bash
npx @peril-ai/mcp --port 3001
```

Or programmatically:

```typescript
import { createMCPServer } from "@peril-ai/mcp";

const server = createMCPServer({
  port: 3001,
  perilEndpoint: "http://localhost:3000/api",
});

await server.start();
```

## Agent Configuration

Add Peril's MCP server to your agent's config file. The config tells your agent
how to launch the Peril MCP server and connect to it.

### Claude Code

Create or edit `.mcp.json` in your project root:

```json
{
  "mcpServers": {
    "peril": {
      "command": "npx",
      "args": ["@peril-ai/mcp"]
    }
  }
}
```

### Claude Desktop

Add to `~/.claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "peril": {
      "command": "npx",
      "args": ["@peril-ai/mcp", "--port", "3001"]
    }
  }
}
```

### Other MCP-Compatible Agents

Any agent that supports the Model Context Protocol can connect to Peril. The
MCP server uses stdio transport by default — just point your agent at:

```bash
npx @peril-ai/mcp
```

Set `PERIL_SERVER_URL` if the backend runs on a non-default port:

```bash
PERIL_SERVER_URL=http://localhost:3001 npx @peril-ai/mcp
```

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `port` | `number` | `3001` | MCP server port |
| `perilEndpoint` | `string` | `http://localhost:3000/api` | Peril REST API URL |
| `transport` | `"stdio" \| "http"` | `"stdio"` | Transport protocol |

<Callout variant="note">
  Most MCP clients use stdio transport by default. Use HTTP transport
  only when running the MCP server as a standalone service.
</Callout>
