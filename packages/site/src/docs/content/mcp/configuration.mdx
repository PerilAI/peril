# MCP Configuration

The Peril MCP server exposes review tasks to coding agents using the
Model Context Protocol.

## Starting the Server

```bash
npx peril-mcp
```

The shipped MCP server uses stdio transport and reads from
`PERIL_SERVER_URL` when you need a non-default backend URL.

Or use the package programmatically:

```typescript
import { PerilMcpServer, startStdioServer } from "@peril-ai/mcp";

startStdioServer({
  serverUrl: "http://127.0.0.1:4173",
  resolvedBy: "codex"
});

const server = new PerilMcpServer({
  serverUrl: "http://127.0.0.1:4173"
});

const reviews = await server.callTool("list_reviews", {
  status: "open"
});
```

## Agent Configuration

Add Peril's MCP server to your agent's config file. The config tells your agent
how to launch the Peril MCP server and connect to it.

### Claude Code

Create or edit `.mcp.json` in your project root:

```json
{
  "mcpServers": {
    "peril": {
      "command": "npx",
      "args": ["peril-mcp"]
    }
  }
}
```

### Other MCP-Compatible Agents

Any agent that supports the Model Context Protocol can connect to Peril. The
MCP server uses stdio transport by default — just point your agent at:

```bash
npx peril-mcp
```

Set `PERIL_SERVER_URL` if the backend runs on a non-default port:

```bash
PERIL_SERVER_URL=http://127.0.0.1:3000 npx peril-mcp
```

<Callout variant="tip">
  We use the bin name `peril-mcp` instead of the package name `@peril-ai/mcp`
  so that npx can resolve it from parent `node_modules` in monorepos.
</Callout>

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `serverUrl` | `string` | `http://127.0.0.1:4173` | Base Peril server URL used for REST requests |
| `resolvedBy` | `string` | `@peril-ai/mcp` | Actor recorded when the MCP server resolves a review |

<Callout variant="note">
  `startStdioServer()` is the supported transport entrypoint today. Use
  `PerilMcpServer` directly when you want in-process tool calls for tests or
  custom integrations.
</Callout>
