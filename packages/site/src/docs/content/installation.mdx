# Installation

Peril is distributed as a set of npm packages. Install the ones you need for
your workflow.

## Agent-Assisted Installation

The easiest way to install Peril is with the `peril-setup` skill. It detects
your framework, installs the right packages, configures MCP, and verifies the
setup.

```bash
npx skills add PerilAI/peril
```

Then tell your agent: _"Set up Peril in this project"_.

<Callout variant="tip">
  The skill automatically detects React, Vue, Svelte, and vanilla JS projects
  and installs only the packages you need. It also handles MCP configuration
  for your specific agent.
</Callout>

## Manual Package Installation

### Package Overview

```bash
# Core SDK (required)
npm install @peril-ai/core

# React adapter (for React apps)
npm install @peril-ai/react

# Local server (for development)
npm install @peril-ai/server

# MCP server (for agent integration)
npm install @peril-ai/mcp
```

Or install everything at once:

```bash
npm install @peril-ai/core @peril-ai/react @peril-ai/server @peril-ai/mcp
```

For non-React apps, skip `@peril-ai/react` and use `@peril-ai/core` directly — see
[Core SDK Overview](/docs/core/overview) for the vanilla JS API.

## Requirements

- **Node.js** 18.0 or later
- **TypeScript** 5.0+ (recommended but not required)
- **React** 18+ (for `@peril-ai/react` only)

## Configuration

Peril uses sensible defaults, but you can customize behavior via the
`ReviewProvider` props or MCP/server options:

```tsx
<ReviewProvider
  serverUrl="http://127.0.0.1:4173"
  showToggle
  reviewerName="Design QA"
/>
```

`serverUrl` is the Peril server base URL. The React adapter submits reviews to
`${serverUrl}/api/reviews`. If you omit it, the browser will post to the
current origin's `/api/reviews` path.

## Storage

By default, the Peril server stores reviews in a `.peril/` directory:

```
.peril/
  reviews/
    rev_<ulid>/
      review.json     # Full review object
      element.png     # Element screenshot
      page.png        # Page screenshot
  index.json          # Review index for fast queries
```

<Callout variant="warning">
  Add `.peril/` to your `.gitignore` to avoid committing review artifacts
  to version control.
</Callout>
