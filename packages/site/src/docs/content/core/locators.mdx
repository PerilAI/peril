# Locators

Locators are the bridge between human feedback and agent action. Peril generates
multiple locator strategies for each element, giving agents the best chance of
finding the right element programmatically.

## Locator Bundle

Each review includes a locator bundle ordered by resilience:

```typescript
interface LocatorBundle {
  testId?: string;    // data-testid="submit-btn"
  role?: {
    type: string;
    name: string;
  };
  css: string;        // .form-actions > button.primary
  xpath: string;      // /html/body/div/form/button[2]
  text?: string;      // "Submit Order"
}
```

## Strategy Priority

| Priority | Strategy | Resilience | Notes |
|----------|----------|-----------|-------|
| 1 | `testId` | Highest | Stable across refactors |
| 2 | `role` | High | ARIA-based, semantic |
| 3 | `css` | Medium | May break on class changes |
| 4 | `xpath` | Low | Fragile to DOM structure |
| 5 | `text` | Lowest | Content may change |

## Helper Functions

```typescript
import {
  generateLocatorBundle,
  getBestLocatorSummary,
  getRankedLocators
} from "@peril-ai/core";

const bundle = generateLocatorBundle(element);
const rankedLocators = getRankedLocators(bundle);
const summary = getBestLocatorSummary(bundle);
```

<Callout variant="note">
  `generateLocatorBundle()` follows the same priority order documented in the
  data model: `testId > role > css > xpath > text`.
</Callout>
