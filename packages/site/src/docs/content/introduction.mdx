# Introduction

Peril turns visual UX feedback into structured, agent-executable tasks via MCP.
A reviewer selects a UI element in a live app, adds a comment, and a coding agent
receives locators, screenshots, and structured context to fix the issue without
human re-description.

## How It Works

1. **Click** — A reviewer clicks any element in your app while review mode is active
2. **Comment** — They describe the issue, and Peril captures element locators, a screenshot, and DOM context
3. **Ship** — A coding agent picks up the structured task via MCP and fixes the issue

## Packages

| Package | Description |
|---------|-------------|
| `@peril-ai/core` | Framework-agnostic capture SDK — overlay, locators, serialization |
| `@peril-ai/react` | React adapter — `ReviewProvider`, `useReviewMode` |
| `@peril-ai/server` | Local dev server with REST API, storage, and dashboard |
| `@peril-ai/mcp` | MCP server that exposes review tasks to coding agents |

## Quick Example

```tsx
import { ReviewProvider, useReviewMode } from "@peril-ai/react";

function App() {
  return (
    <ReviewProvider serverUrl="http://127.0.0.1:4173" showToggle>
      <YourApp />
      <ReviewToggle />
    </ReviewProvider>
  );
}

function ReviewToggle() {
  const { enabled, setEnabled } = useReviewMode();
  return (
    <button onClick={() => setEnabled((current) => !current)}>
      {enabled ? "Exit Review" : "Start Review"}
    </button>
  );
}
```

<Callout variant="tip">
  Peril is designed for local development workflows. Point your MCP-compatible
  agent at the local MCP server and it can retrieve structured tasks from your
  local Peril backend.
</Callout>
