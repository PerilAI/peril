# React Setup

`@peril-ai/react` provides React-specific bindings for Peril, including a context
provider and hooks for managing review mode.

## Installation

```bash
npm install @peril-ai/react @peril-ai/core
```

## Provider Setup

Wrap your application (or the section you want to review) with `ReviewProvider`:

```tsx
import { ReviewProvider } from "@peril-ai/react";

function App() {
  return (
    <ReviewProvider
      serverUrl="http://127.0.0.1:4173"
      showToggle
      reviewerName="QA"
    >
      <YourApp />
    </ReviewProvider>
  );
}
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `serverUrl` | `string` | `undefined` | Base Peril server URL. Reviews post to `${serverUrl}/api/reviews`. |
| `initialEnabled` | `boolean` | `false` | Start with review mode enabled. |
| `showToggle` | `boolean` | `false` | Render the built-in floating toggle button. |
| `reviewerName` | `string` | `undefined` | Include a reviewer name in review metadata. |
| `keyboardShortcut` | `ReviewKeyboardShortcut \| false` | Default shortcut | Customize or disable the keyboard shortcut. |
| `submitOptions` | `Omit<SubmitReviewOptions, "endpoint">` | `{}` | Override retry, fetch, header, or abort behavior for submissions. |
| `children` | `ReactNode` | Required | App content |

<Callout variant="warning">
  The `ReviewProvider` must be mounted inside your app's root. It adds a
  transparent overlay when review mode is active — this does not affect
  your app's layout or event handling in normal mode.
</Callout>

Additional callbacks such as `onReviewCreated`, `onReviewError`, `onHover`,
`onSelect`, and `onToggleRequest` are available when you need to integrate the
overlay with your own app state.
