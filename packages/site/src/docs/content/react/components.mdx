# React Components

`@peril-ai/react` currently ships one React component plus a state hook for
controlling it.

## `ReviewProvider`

The root provider that mounts the review overlay, handles selection, captures
artifacts, and submits reviews to the Peril server.

```tsx
import { ReviewProvider } from "@peril-ai/react";

function App() {
  return (
    <ReviewProvider serverUrl="http://127.0.0.1:4173" showToggle>
      <YourApp />
    </ReviewProvider>
  );
}
```

<Callout variant="tip">
  Most apps only need `ReviewProvider` plus `useReviewMode()` for a custom
  toggle or status display.
</Callout>

## `useReviewMode()`

Use the hook when you want to render your own controls instead of relying on the
built-in toggle button:

```tsx
import { useReviewMode } from "@peril-ai/react";

function ReviewToggle() {
  const { enabled, setEnabled } = useReviewMode();

  return (
    <button onClick={() => setEnabled((current) => !current)}>
      {enabled ? "Exit Review" : "Start Review"}
    </button>
  );
}
```

## Helper Re-exports

The React package also re-exports `getBestLocatorSummary`, `getRankedLocators`,
and `locatorPriority` from `@peril-ai/core` so React apps can use the same
locator helpers without adding a second import path.
