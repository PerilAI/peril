# Quick Start

Get up and running with Peril in under 5 minutes.

## Prerequisites

- Node.js 18+
- A web application (React, Vue, Svelte, vanilla JS, etc.)
- A coding agent (Claude Code, Cursor, Codex CLI, Gemini CLI, OpenCode, etc.)

## Option A: Agent-Assisted Setup (Recommended)

The fastest way to set up Peril is to let your coding agent do it. Download the
`peril-setup` skill and ask your agent to configure everything for you.

```bash
# Add the skill as a Claude Code slash command
curl -sL https://github.com/PerilAI/peril/raw/main/skill/peril-setup/SKILL.md \
  -o .claude/commands/peril-setup.md --create-dirs
```

Then tell your agent:

```
Set up Peril in this project so my team can annotate UI bugs
```

Or in Claude Code, run `/peril-setup` directly.

<Callout variant="tip">
  The setup skill handles everything — installing packages, wrapping your app,
  configuring MCP, adding `.peril/` to `.gitignore`, and verifying the setup.
  It works with React, Vue, Svelte, and vanilla JS projects.
</Callout>

## Option B: Manual Setup

If you prefer to set things up yourself, follow these steps.

### 1. Install

```bash
npm install @peril/core @peril/react @peril/server @peril/mcp
```

For non-React apps, skip `@peril/react` and use `@peril/core` directly.

### 2. Add the Provider

Wrap your app with `ReviewProvider`:

```tsx
import { ReviewProvider } from "@peril/react";

function App() {
  return (
    <ReviewProvider endpoint="http://localhost:3000/api">
      <YourApp />
    </ReviewProvider>
  );
}
```

### 3. Start the Server

```bash
npx peril-server --port 3000
```

### 4. Connect Your Agent

Add Peril's MCP server to your agent's configuration:

```json
{
  "mcpServers": {
    "peril": {
      "command": "npx",
      "args": ["@peril/mcp", "--port", "3001"]
    }
  }
}
```

See [MCP Configuration](/docs/mcp/configuration) for agent-specific config file locations.

### 5. Start Reviewing

1. Open your app in the browser
2. Press `Ctrl+Shift+R` to activate review mode
3. Click any element and leave a comment
4. Your agent will receive the structured task via MCP
